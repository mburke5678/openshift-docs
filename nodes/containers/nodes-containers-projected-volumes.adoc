= Mapping volumes using projected volumes
include::_attributes/common-attributes.adoc[]

toc::[]





[role="_abstract"]
You can  consolidate multiple data sources into a single directory for your pods, by using a projected volume. A projected volume allows you to manage secrets, config maps, and other volume sources collectively within the same mount point.

You can project any of the the following types of volume sources:

* Secrets

* Config Maps

* Downward API

[NOTE]
====
All sources are required to be in the same namespace as the pod.
====
